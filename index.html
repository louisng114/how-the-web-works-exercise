Part One: Solidify Terminology

In your own terms, define the following terms:

  • What is HTTP?

    • Short for Hypertext Transfer Protocol. It is the protocol to communicate with servers.

  • What is a URL?

    • The language-based address of a website corresponded to IP addresses.

  • What is DNS?

    • Short of Domain Name System. It is the way the internet translate between URL and IP addresses.

  • What is a query string?

    • Part of an URL. It is located at the end of an URL for additional parameters.

  • What are two HTTP verbs and how are they different?

    • Two HTTP verbs are GET and POST. THey are both requests. GET takes information and does not change data on the server side. POST sends data to a server.

  • What is an HTTP request?

    • Something the user asks a server to do.

  • What is an HTTP response?

    • The response from a HTTP request

  • What is an HTTP header? Give a couple examples of request and response headers you have seen.

    • Specific details of a request/response. Some examples of request headers are authority, method, and accept-language. Some examples of response headers are content-type, date, status, and server.

  • What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser?

    • The browser translates the URL into and IP address with the DNS. A request for the content is sent to the server with the IP address. The server decides what to do and sends back a response. Base on the response, the browser displays a web page.
